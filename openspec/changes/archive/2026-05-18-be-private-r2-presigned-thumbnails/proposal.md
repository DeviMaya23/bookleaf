## Why

Thumbnail URLs are currently built from a public R2 bucket URL, meaning anyone who discovers the URL pattern (`<publicURL>/users/<userID>/thumbnails/<imageID>.jpg`) can access images directly from R2 without going through the app. Making the bucket private and switching thumbnails to presigned GET URLs closes this: objects are only accessible via time-limited URLs generated by the backend for authenticated users.

## What Changes

- Remove `CDNUrl` from `StorageService` interface and `r2Storage` implementation
- Remove `PublicURL` from `r2Storage`, `R2Config`, and config loading (`R2_PUBLIC_URL` env var)
- Add a private `thumbnailURL(ctx, path *string) *string` helper on `imageUsecase` — generates a presigned GET URL for a thumbnail path, returns nil if path is nil or presigning fails
- Move thumbnail URL generation into the usecase layer (all 5 affected methods: `ListImages`, `ListTrashed`, `GetImage`, `Restore`, `UpdateImage`) — handler receives URLs ready to render
- Add `ThumbnailURL *string` to `ImageDetail`
- Introduce `ImageItem{Image *domain.Image, ThumbnailURL *string}` as the return type for list and mutation results — `ListImagesResult`, `ListTrashedResult`, `Restore`, and `UpdateImage` all switch to this type
- `toImageResponse` in the handler becomes a plain field mapper with no storage dependency — no longer a method on `ImageHandler`

## Impact

- `internal/storage/storage.go` — remove `CDNUrl` from interface
- `internal/storage/r2.go` — remove `CDNUrl` method, remove `publicURL` field, remove `PublicURL` from constructor
- `internal/config/config.go` — remove `PublicURL` from `R2Config`, remove `R2_PUBLIC_URL` env var loading
- `internal/usecase/image_usecase.go` — add `thumbnailURL` helper; update `ListImages`, `ListTrashed`, `GetImage`, `Restore`, `UpdateImage` return types; add `ImageItem` type
- `internal/handler/image.go` — remove `store` field from `ImageHandler`; `toImageResponse` becomes a package-level function accepting `ImageItem`; update all 5 call sites
- `.env.example` — remove `R2_PUBLIC_URL`
- No new dependencies
- No frontend changes required
